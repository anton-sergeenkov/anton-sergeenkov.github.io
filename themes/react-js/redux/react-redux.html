React-Redux

npm i --save redux npm i --save react-redux Документация

1. createStore redux

  • createStore - создание store
  • createStore
    • ☰ reducer - функция-обработчик изменений (редьюсер)
    • initialState - начальное состояние
    • middleware - усилители (н-р: логгер (logger), который просто пишет в консоль все что происходит с наблюдаемым объектом)

2. combineReducers redux

  • combineReducers - объединение нескольких reducer

3. Provider react-redux

  • <Provider /> - рутовый компонент для того чтобы прокидывать Store. Необходим для того чтобы не пробрасывать Store в каждый компонент самостоятельно. Store будет доступен в каждом дочернем компоненте Provider из функции connect

4. Connect react-redux

  • connect - подключение компонента к Store (декоратор)
  • Результат работы функции connect - новый присоединенный компонент, который оборачивает переданный компонент. У нас был компонент <App />, а на выходе получился <Connected(App)>
  • В свойствах (props) компонента <App /> теперь есть метод redux store - dispatch, и объект свойств. Это так же результат работы функции connect

connect()(Component)                                    // Subscribe=False Action=False
connect(mapStateToProps)(Component)                     // Subscribe=True  Action=False
connect(null, mapDispatchToProps)(Component)            // Subscribe=False Action=True
connect(mapStateToProps, mapDispatchToProps)(Component) // Subscribe=True  Action=True
